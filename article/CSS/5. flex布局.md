# flex 布局

## 什么是 flex 布局

`flex` 是 `Flexible Box` 的缩写，意为**弹性布局**。可以让我们盒状态模型更加灵活。

所有的容器都可以指定为 `flex` 布局。

```css
.box {
  display: flex;
}
```

当我们设置 `flex` 容器时，所有子元素自动成为 `flex` 的容器成员，称为 `flex` 项目。

![image](https://user-images.githubusercontent.com/66205484/147448675-c2eca74c-fbc7-4fb6-9127-940a3de0ac6d.png)

而在 `flex` 布局中，有两根轴线：主轴(`main`) 和 交叉轴(`cross`)。

图中已经标明了，默认情况下的主轴指向为横向从左向右，而交叉轴默认为纵向从上到下。

## flex 容器属性

以下六个为 `flex` 容器属性，用来调整容器内部布局。

- flex-direction
- flex-wrap
- flex-flow
- justify-content
- align-items
- align-centent

### flex-direction

`flex-direction` 属性决定了**主轴的方向**。

```css
.box {
  flex-direction: row | row-reverse | column | column-reverse;
}
```

我们来对四个值做一个解释。

- row(默认值): 主轴水平，起点在左
- row-reverse: 主轴水平，起点在右
- column: 主轴纵向，起点在上
- column-reverse: 主轴纵向，起点在下

我们来看一下具体的图

![image](https://user-images.githubusercontent.com/66205484/147451858-33d52d01-6946-4909-9454-c392d6b6a0f7.png)

可以自行去验证一下。

### flex-wrap

`flex-wrap` 属性用来设置在轴线上是否进行换行。

```css
.box {
  flex-wrap: nowrap | wrap | wrap-reverse;
}
```

- nowrap(默认值): 不换行
- wrap: 换行
- wrap-reverse: 换行，并且倒序

设置 `nowrap` 时，元素宽度尽管设置了，也会被进行压缩，导致宽度减少。

![image](https://user-images.githubusercontent.com/66205484/147454089-270e470f-5cdf-4460-a1d8-012d10073fb7.png)

### flex-flow

`flex-flow` 属性就是 `flex-direction` 和 `flex-wrap` 两个属性的简写方式，默认值为 `row nowrap`。

```css
.box {
  flex-flow: row nowrap;
}
```

### justify-content

`justify-content` 是用来设置项目在主轴方向上的对齐方式。

```css
.box {
  justify-content: flex-start | flex-end | center | space-between | space-around;
}
```

一共有五个可选取值。

- flex-start(默认值): 主轴起点对齐
- flex-end: 主轴终点对齐
- center: 主轴中间点对齐
- space-between: 主轴起点终点两侧贴边，中间项目间隔开，且间隔相等
- space-around: 主轴起点终点两侧的间隔相等，且大小是中间间隔的一半，而中间项目之间的间隔相等

![image](https://user-images.githubusercontent.com/66205484/147520609-16322082-8e44-4d97-9533-4a230261b213.png)

在图中展示的是 `flex-direction: row` 的情况，具体布局要根据具体的主轴设置方向来定。

### align-items

`align-items` 是用来设置项目在交叉轴方向上的对齐方式。

```css
.box {
  align-items: flex-start | flex-end | center | space-between | space-around;
}
```

一共有五个可选取值。

- flex-start(默认值): 交叉轴起点对齐
- flex-end: 交叉轴终点对齐
- center: 交叉轴中间点对齐
- baseline: 项目第一行文字的基线对齐
- stretch(默认值): 如果项目未设置高度或设置 `auto`，则占满整个容器高度

![image](https://user-images.githubusercontent.com/66205484/147521535-9acc12ca-ce52-4a17-88d3-767484f784d5.png)

该图是根据默认的 `flex-direction: row` 来显示，具体的还需要根据方向来表示。

### align-centent

`align-content` 是用来设置多根轴线的对齐方式，如果只有一根轴线，则不起作用。

```css
.box {
  align-items: flex-start | flex-end | center | space-between | space-around;
}
```

一共六个可选值。

- flex-start: 交叉轴的起点对齐
- flex-end: 交叉轴终点对齐
- center: 交叉轴中间点对齐
- space-between: 交叉轴的两端贴边，项目之间的间隔均匀分布
- space-around: 交叉轴两端的间隔相等，而项目之间的间隔是两边间隔的两倍
- stretch(默认值): 轴线占满整个交叉轴

![image](https://user-images.githubusercontent.com/66205484/147523075-4a66f2b7-ec00-4c1e-afa0-b2ad97cbe9a1.png)
我们也是按照 `flex-direction: row` 来的，具体的还需要根据方向来表示。

## 项目的属性

- order
- flex-grow
- flex-shrink
- flex-basis
- flex
- align-self

### order

`order` 是在项目中设置的属性，而设置该属性，可以调整项目的排列顺序，数值越小，排列在越前面，默认为 `0`。

```css
.item {
  order: 1; /* 可以为任意数字 */
}
```

我们画一个简单的图，来表示 `order`。

![image](https://user-images.githubusercontent.com/66205484/147517419-fd74ae88-68ca-498b-84a0-5b7fe9dae36b.png)

相同值会按照设置顺序排列，而不同值则按照从小到大排列。

### flex-grow

`flex-grow` 属性定义了项目的放大比例，默认为 `0`，如果空间又剩余，也不放大。

我们来看图，举一下例子。

![image](https://user-images.githubusercontent.com/66205484/147517721-462b7b5f-338a-48bd-944d-17ea1b240f09.png)

- 第一个图，我们可以看到，所有的 `flex-grow` 都给了默认值 `0`, 且给每一个 `item` 设置不同宽度时显示，所以并不会进行放大，只会展示原来大小。

- 第二个图是，我们全部设置为 `1` 时，如果 `item` 没有设置宽度，会均匀平分宽度，而如果有设置宽度，则会保留宽度差进行放大。

- 第三个图是，我们一个设置为 `2`，其他均为 `1` 时，如果没设置宽度，会以 `1:2:1` 来平分宽度，而如果有宽度设置，也会存在部分偏差。

所以为了准确性，最好不设置 `item` 宽度。

### flex-shrink

`flex-shrink` 属性定义了项目的缩小比例，默认为 `1`，如果空间不足，该项目会缩小。

```css
.item {
  flex-fhrink: 1;
}
```

当我们有设置为 `0` 的时候，则项目不会进行压缩，会保留原来设置的大小。

![image](https://user-images.githubusercontent.com/66205484/147518425-609714d8-127c-4d81-9d0b-eaa33c22d652.png)

我们可以看上面的 `nowrap` 的情况，不够大小时，进行了压缩，而这时候，我们将其中一个设置为 `flex-shrink: 0`，则不会对这个 `item` 进行压缩。

也许不明确，我们来用代码举个例子吧

```css
.box {
  width: 500px;
  height: 100px;
  background-color: #efcca1;
  display: flex;
}
.item {
  width: 100px;
  height: 50px;
  background-color: #1890ff;
  border: 1px solid black;
}
```

```html
<div class="box">
  <div class="item">1</div>
  <div class="item">2</div>
  <div class="item">3</div>
  <div class="item">4</div>
  <div class="item">5</div>
  <div class="item">6</div>
</div>
```

![image](https://user-images.githubusercontent.com/66205484/147518704-6a2abfa2-e425-4076-8872-74ba71bf24b4.png)

这时候的显示，我们可以用开发者工具查看大小不足 `100px`，已经被压缩了。

我们将第二个，添加 `flex-shrink: 0`，我们在来看看结果。

```html
<div class="child" style="flex-shrink: 0">2</div>
```

![image](https://user-images.githubusercontent.com/66205484/147518832-740ccb32-fc8c-4429-a0f5-353dda075ed8.png)

发现，第二个的宽度就不被压缩了，保留了原来的宽度。

### flex-basis

`flex-basis` 属性定义了分配多余空间之前，项目占据的主轴空间。默认值为 `auto`，即项目的本来大小。

```css
.item {
  flex-basis: auto;
}
```

可以设置 `px` 值，来指定项目将占据的固定空间大小。

我们还是按照 `flex-shrink` 中使用的代码来举例子，我们设置 `flex-basis: 50px`，我们会发现项目的本身大小就被压缩到 `50px` 了，而不是我们设置的 100px，或者

```css
.item {
  width: 100px;
  height: 50px;
  background-color: #1890ff;
  border: 1px solid black;
  flex-basis: 50px;
}
```

![image](https://user-images.githubusercontent.com/66205484/147519312-9d1a2f85-53f6-4fa5-939b-f4b32d10de56.png)

### flex

`flex` 属性是 `flex-grow`，`flex-shrink`，`flex-basis` 三个属性的缩写，默认值为 `0 1 auto`

```css
.item {
  flex: none | flex-grow flex-shrink flex-basis;
}
```

快截止有：`auto`(`1 1 auto`) 和 `none`(`0 0 auto`)

`flex: 1` 是我们经常会在开发中用到的，而 `flex: 1` 代表着什么呢？其实就是 `flex: 1 1 0`。

`flex-shrink` 与 `flex-basis` 省略时，默认分别为 `1` 和 `0`。

### align-self

`align-self` 属性允许单个 `item` 可以有其他项目不一样的对齐方式，可以覆盖容器属性中 `align-items` 属性，默认值为 `auto`。

其值是与 `align-items` 属性相同。

```css
.item {
  align-self: auto | flex-start | flex-end | center | baseline | stretch;
}
```
